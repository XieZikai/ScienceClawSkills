# COF Remote Optimizer

Run COF structure relaxations in two cascaded stages: (1) coarse DFTB+ minimization to remove large strains, then (2) MACE force-field refinement after a light coordinate perturbation. This skill is the hub that orchestrates the previously delivered `dftb`, `file-upload`, and `mace-cof-optimizer` skills plus the helper scripts in `scripts/`.

## Dependencies
- **Input**: CIFs generated by the `dimer-mol2cif` skill (aldehyde/amine pairings)
- **Coarse stage**: `file-upload` + `dftb` skills, script-runner endpoints configured in `skills/dftb/scripts/config/dftb_config.json`
- **Fine stage**: `mace-cof-optimizer` skill with `scorer_url` configured in `skills/mace-cof-optimizer/scripts/config/mace_optimizer_config.json`
- **Helper scripts** (in this skill):
  - `scripts/pack_dftb_zip.py` – bundle CIF + POSCAR + `dftb_in.hsd`
  - `scripts/extract_dftb_result.py` – turn DFTB geo_end.* output into `*_dftb_opted.cif`
  - Reference notebooks/files for perturbation + local logic live under `resources/`

## Workflow at a glance
1. **Bundle DFTB inputs** for each CIF with `pack_dftb_zip.py` (auto-generates POSCAR from CIF).
2. **Upload each ZIP** via the `file-upload` skill → obtain remote `file_url`.
3. **Submit & poll DFTB jobs** using the `dftb` skill (`dftb_client.py submit` / `poll`).
4. **Convert DFTB artifacts** (`geo_end.gen/xyz`) into `*_dftb_opted.cif` using `extract_dftb_result.py`.
5. **Apply coordinate perturbations** (Step 3.1 logic) so the MACE stage escapes identical local minima.
6. **Batch-submit perturbed CIFs** to the remote MACE scorer via `mace_cof_optimizer/scripts/mace_cof_client.py`.
7. **Download final CIFs** from the scorer and archive alongside metadata (density/energy JSON, CSV summary).

## Step 1 – DFTB+ coarse relaxation
1. **Stage inputs**
   - Copy or symlink the starting CIFs into a working folder, e.g. `runs/20260323/dftb_inputs/`.
   - Ensure an appropriate `dftb_in.hsd` (template under `resources/dftb_opt/dftb_in.hsd`). Edit the lattice/SCF knobs there if this batch requires different parameters.
2. **Build submission bundles**
   ```bash
   cd skills/cof-remote-optimizer
   python scripts/pack_dftb_zip.py ../path/to/cifs/*.cif \
     --hsd resources/dftb_opt/dftb_in.hsd \
     --out runs/20260323/dftb_zips
   ```
   - The script re-generates POSCAR with ASE to avoid stale files.
3. **Upload each ZIP**
   - Use the `file-upload` skill (see its SKILL) to push `*_dftb_input.zip`. Capture the returned `file_url` per structure.
4. **Submit remote jobs**
   ```bash
   cd skills/dftb/scripts
   ./dftb_client.py --config config/dftb_config.json submit <file_url> --tasks-per-node 8 --nodes 1
   ```
   - Record the `execution_id`. Repeat per ZIP (a shell loop is fine; avoid overloading the queue).
5. **Monitor completion**
   ```bash
   ./dftb_client.py --config config/dftb_config.json poll <execution_id>
   ```
   - The poller prints `result_assets` with HTTPS links (already rewritten). Download each artifact locally, e.g. under `runs/20260323/dftb_results/<stem>.zip`.
6. **Convert to CIF**
   ```bash
   cd skills/cof-remote-optimizer
   python scripts/extract_dftb_result.py runs/20260323/dftb_results/<stem>.zip --out runs/20260323/dftb_opted
   ```
   - Outputs follow `*_dftb_opted.cif` naming; these feed the MACE stage. Validation tip: compare lattice parameters vs. the provided reference `resources/dftb_opt/test_dftb_opted.cif`.

### Notes / troubleshooting
- If the DFTB script returns multiple `geo_end.*` variants, the extractor picks `geo_end.gen` first, `geo_end.xyz` second. Override inside the script if your runner changes naming.
- Keep uploads and downloads serialized if the remote object store rate-limits (HTTP 429). The `file-upload` skill supports exponential backoff—reuse it instead of hand-written curl.

## Step 2 – MACE refinement
1. **Micro-perturbation (Step 3.1 logic)**
   - Goal: remove redundant sym-equivalent structures and add slight +/- Z displacements before the MACE relaxation.
   - Reference: `resources/mace_opt/1-dehydro-Z-axis-permutation.ipynb`. When converting to script/CLI, mirror this order:
     1. Strip hydrogens that sit at the same fractional coordinates as heavy atoms (consistent with the notebook logic).
     2. Apply deterministic ±Δz shifts (few 0.01 Å) per atom to diversify starting points.
     3. Write perturbed CIFs into `dftb_opted_cifs_perturbed/`.
   - Until a CLI is authored, run the notebook once per batch (Jupyter, VS Code, or `jupyter nbconvert --execute`). Document perturbation amplitude in your run log.
2. **Prepare MACE payload folder**
   - Place all perturbed CIFs inside e.g. `runs/20260323/mace_inputs/`. Keep filenames stable; the scorer sorts them before batching.
3. **Submit to the scorer**
   ```bash
   cd skills/mace-cof-optimizer/scripts
   ./mace_cof_client.py --config config/mace_optimizer_config.json \
     submit /absolute/path/to/runs/20260323/mace_inputs job_20260323
   ```
   - The client consolidates CIF text into a single JSON (`get_opt_only_input_json`).
4. **Poll and retrieve**
   ```bash
   ./mace_cof_client.py --config config/mace_optimizer_config.json poll <job_id>
   ./mace_cof_client.py --config config/mace_optimizer_config.json cifs --job-id <job_id> --out runs/20260323/mace_results
   ./mace_cof_client.py --config config/mace_optimizer_config.json result --job-id <job_id> > runs/20260323/mace_results/job.json
   ```
   - The `cifs` subcommand writes `structure_###.cif`. Rename/match them back to input stems immediately (e.g., keep a manifest mapping order).
5. **Collect metadata**
   - The remote JSON mirrors the local `2_opt_MACE.py` outputs. When needed, rebuild the CSV summary locally for provenance.

### Recommended directory layout
```
runs/<date>/
  dftb_inputs/
  dftb_zips/
  dftb_results/
  dftb_opted/
  mace_inputs/        # perturbed CIFs
  mace_results/
    final_cifs/
    job.json
```
This keeps every artifact traceable across both stages.

## Validation checklist
- Lattice drift between starting CIF and `*_dftb_opted.cif` <5 %; otherwise re-run with tighter DFTB thresholds.
- MACE stage should reduce forces to fmax ≤0.01 eV/Å. Confirm via scorer metadata; requeue if `status != success` or if any structure reports fallback defaults (density=1e5 in the legacy script).
- Final CIFs go into downstream pipelines (e.g., adsorption screening). Archive both intermediate and final CIFs in `outputs/` for reproducibility.
